# 費用申請台帳

会社への費用申請を、**申請未 → 申請済み → 受取済み** の3段階で追いかけるための台帳アプリです。
期（4月1日〜翌3月31日）ごとの上限額に対して、いま何円使えるかが常に分かります。

依存ライブラリなし。`index.html` 1枚だけで動きます。

## 使う

**https://<username>.github.io/keihi-daicho/**

ローカルで開く場合は `index.html` をブラウザにドラッグするだけです。

## できること

### 期ごとの枠管理

期は **4月1日から翌3月31日** です。利用日を入れると自動でその期に振り分けられるので、1〜3月に使った分が前年度の期に入るような年度またぎでも迷いません。右上の ◀ ▶ とプルダウンで他の期も見られます。

上限は初期値30万円で、画面右上の「上限」欄から変更できます。残枠が上限の1割を切ると警告、超えると赤字で超過額を表示します。

### 3段階のステータス

| ステータス | 意味 |
| --- | --- |
| 申請未 | 使ったが、まだ会社に出していない |
| 申請済み | 出したが、まだ入金されていない |
| 受取済み | 完了 |

各行の「申請した →」「受け取った →」ボタンで次の段階に送られます。押し間違えても「戻す」で1段階戻せます。

3つとも枠を消費するものとして計算しているので、**残枠 = 上限 −（申請未 + 申請済み + 受取済み）** です。これから申請する分まで織り込んだ数字になります。

### 小項目の内訳

1件の費用を **参加費・宿泊費・交通費・その他** に分けて記録できます。全部埋める必要はなく、書籍代だけなら「その他」に入れるだけで済みます。合計は入力しながら自動で加算されます。

サマリーの「この期の内訳」には費目ごとの期合計が出るので、今期の交通費がいくらかがひと目で分かります。

## データの保存先

入力したデータは**ブラウザの localStorage に保存されます**。サーバーには一切送信されないので、他人から見られることはありません。

裏を返すと、**別のブラウザや別の端末とは自動で同期しません**。移すときは画面下部の「バックアップを保存」でJSONを書き出し、移行先で「バックアップから復元」してください。ブラウザのデータを消すと台帳も消えるので、区切りのいいところでバックアップを取っておくのが安全です。

保存キーは以下の2つです。

- `keihi-daicho-v1` … 台帳データ本体
- `keihi-theme` … ライト／ダークの選択（未設定ならOSの設定に追従）

## 開発

ビルド手順はありません。`index.html` を直接編集します。中身は上から順に `<style>`、マークアップ、`<script>` の3ブロックです。

データ構造は次のとおりです。

```json
{
  "version": 2,
  "limit": 300000,
  "items": [
    {
      "id": "k...",
      "date": "2026-05-12",
      "name": "技術カンファレンス 参加",
      "amounts": { "fee": 55000, "lodge": 28000, "trans": 31400, "other": 3200 },
      "amount": 117600,
      "note": "領収書は Dropbox/2026/conf に保存",
      "status": "applied",
      "appliedAt": "2026-05-20"
    }
  ]
}
```

`amount` は `amounts` の合計を保持する派生値で、読み込み時に必ず再計算されます。小項目を持たない v1 のデータは、読み込み時に自動で「その他」へ移行します。
