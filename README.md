# 費用申請台帳

会社への費用申請を、**申請未 → 申請済み → 受取済み** の3段階で追いかけるための台帳アプリです。
期（4月1日〜翌3月31日）ごとの上限額に対して、いま何円使えるかが常に分かります。

依存ライブラリなし。`index.html` 1枚だけで動きます。

## 使う

**https://mctoil.github.io/keihi-daicho/**

ローカルで開く場合は `index.html` をブラウザにドラッグするだけです。

## できること

### 期ごとの枠管理

期は **4月1日から翌3月31日** です。利用日を入れると自動でその期に振り分けられるので、1〜3月に使った分が前年度の期に入るような年度またぎでも迷いません。右上の ◀ ▶ とプルダウンで他の期も見られます。

上限は初期値30万円で、画面右上の「上限」欄から変更できます。残枠が上限の1割を切ると警告、超えると赤字で超過額を表示します。

### 3段階のステータス

| ステータス | 意味 |
| --- | --- |
| 申請未 | 使ったが、まだ会社に出していない |
| 申請済み | 出したが、まだ入金されていない |
| 受取済み | 完了 |

各行の「申請した →」「受け取った →」ボタンで次の段階に送られます。押し間違えても「戻す」で1段階戻せます。

3つとも枠を消費するものとして計算しているので、**残枠 = 上限 −（申請未 + 申請済み + 受取済み）** です。これから申請する分まで織り込んだ数字になります。

### 日付

1件につき4つの日付を持てます。**利用日以外はすべて任意**です。

| 欄 | 用途 |
| --- | --- |
| 利用日 | 必須。この日付で期が決まります |
| 期間 開始 / 終了 | WEBアーカイブ受講のように期間があるもの用。期の振り分けには影響しません |
| 支払日 | 実際に費用を支払った日 |
| 申請日 | 会社に申請を出した日 |
| 受取日 | 入金された日 |

**ステータスは申請日・受取日から決まります。** 受取日が入っていれば受取済み、申請日だけなら申請済み、どちらも空なら申請未です。

このため、過去の案件をまとめて登録するときは、記録画面で申請日や受取日を入れるだけで正しいセクションに入ります。ボタンで段階を進めた場合は、空いている日付に今日の日付が自動で入ります（すでに日付が入っていれば上書きしません）。「戻す」を押すと、その段階の日付が消えます。

### 小項目の内訳

1件の費用を **参加費・宿泊費・交通費・その他** に分けて記録できます。全部埋める必要はなく、書籍代だけなら「その他」に入れるだけで済みます。合計は入力しながら自動で加算されます。

サマリーの「この期の内訳」には費目ごとの期合計が出るので、今期の交通費がいくらかがひと目で分かります。

### WEB

オンライン参加・アーカイブ受講のように現地へ行かないものは、記録画面の **WEB** にチェックを入れます。

チェックすると宿泊費と交通費の欄が入力できなくなり、合計からも外れます。金額を入れたあとにチェックした場合はその場で0になるので、合計の変化で何が外れたか分かります。チェックを外せばまた入力できます。

一覧では件名の横に `WEB` バッジが付くので、宿泊費・交通費が無いのは記入漏れではなく元々不要だった、と後から見て判別できます。

### 分割

1つのイベントでも、申請が2回に分かれることがあります。参加費だけ先に申請して受け取り、参加日がまだ先なので交通費と宿泊費は後日あらためて申請する、というような場合です。

このときは明細の「分割」を押します。費目が2つ以上ある明細にだけ出ます。

後から申請する費目にチェックを入れると、残る金額と切り出す金額がその場で表示されます。「分割する」を押すと、選んだ費目が**新しい明細として切り出され、申請未に入ります**。元の明細は残りの費目とステータス・日付をそのまま保ちます。

切り出した側は内容・利用日・期間・WEB・メモを引き継ぎますが、**日付は空**で始まります。これから申請するものだからです。メモは元の明細の内容がそのまま入るので、必要なら編集してください。

両方の明細に `分割 2件` バッジが付き、同じイベント由来だと分かります。切り出した分は申請未に残るので、後日の申請を忘れずに済みます。残枠の計算はどちらも従来どおり枠を消費します。

## データの保存先

入力したデータは**ブラウザの localStorage に保存されます**。サーバーには一切送信されないので、他人から見られることはありません。

裏を返すと、**別のブラウザや別の端末とは自動で同期しません**。移すときは画面下部の「バックアップを保存」でJSONを書き出し、移行先で「バックアップから復元」してください。ブラウザのデータを消すと台帳も消えるので、区切りのいいところでバックアップを取っておくのが安全です。

保存キーは以下の2つです。

- `keihi-daicho-v1` … 台帳データ本体
- `keihi-theme` … ライト／ダークの選択（未設定ならOSの設定に追従）

## 開発

ビルド手順はありません。`index.html` を直接編集します。中身は上から順に `<style>`、マークアップ、`<script>` の3ブロックです。

データ構造は次のとおりです。

```json
{
  "version": 4,
  "limit": 300000,
  "items": [
    {
      "id": "k...",
      "date": "2026-05-12",
      "name": "技術カンファレンス 参加",
      "web": false,
      "amounts": { "fee": 55000, "lodge": 28000, "trans": 31400, "other": 3200 },
      "amount": 117600,
      "periodStart": "",
      "periodEnd": "",
      "paidAt": "2026-05-12",
      "appliedAt": "2026-05-20",
      "receivedAt": "",
      "note": "領収書は Dropbox/2026/conf に保存",
      "status": "applied",
      "groupId": ""
    }
  ]
}
```

`groupId` は分割で作られた明細に付く共通のIDです。空なら分割されていない単独の明細です。

`amount` と `status` はどちらも派生値で、読み込み時に必ず再計算されます。`amount` は `amounts` の合計、`status` は `appliedAt` / `receivedAt` の有無から決まります。`web` が `true` のときは `amounts.lodge` と `amounts.trans` が0に固定されます。

旧バージョンのデータは読み込み時に自動で移行します。小項目を持たない v1 のデータは「その他」へ、日付を持たない v2 のデータは利用日を申請日・受取日に補って、当時の段階を保ちます。v3 のデータは `web: false`、支払日は空として読み込まれます。
